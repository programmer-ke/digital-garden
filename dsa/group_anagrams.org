** Solution

#+begin_src python
from collections import defaultdict

def group_anagrams(input_list: list[str]) -> list[list[str]]:
    index = defaultdict(list)

    for word in input_list:
        key = "".join(sorted(word))
        index[key].append(word)
    return [words for words in index.values()]


strs = ["eat","tea","tan","ate","nat","bat"]
empty = [""]
onlya = ["a"]

print(group_anagrams(strs)) # [["bat"],["nat","tan"],["ate","eat","tea"]]

print(group_anagrams(empty)) # [[""]]

print(group_anagrams(onlya)) # [["a"]]


#+end_src

from collections import defaultdict

def group_anagrams(input_list: list[str]) -> list[list[str]]:
    index = defaultdict(list)

    for word in input_list:
        key = create_key(word)
        index[key].append(word)
    return [words for words in index.values()]

def create_key(word: str) -> tuple[str, ...]:
    key = [0] * 26
    for letter in word:
        index = ord(letter) - ord('a')
        key[index] += 1
    return tuple(key)

This problem belongs to the *string manipulation / hashing / grouping*
category, often called *anagram grouping* or *character frequency

*Key characteristics to identify this type of problem:*

1. *Grouping based on composition* – The output requires collecting
   items that share an underlying property (same letters, same
   frequency of elements, etc.) regardless of order.

2. *Order‑invariant matching* – Two different sequences (words,
   arrays) need to be recognized as equivalent if they contain the
   same multiset of elements. This is the classic "anagram" pattern,
   but it also appears in problems like "group sentences with same
   word frequency" or "group numbers with same prime factors".

3. *Canonical representation* – The core technique is to transform
   each input into a *canonical form* (sorted string, frequency tuple,
   prime product) that is the same for all group members. The
   canonical form acts as a hash key.

4. *Hash map aggregation* – After deriving the canonical key, you
   insert the original item into a dictionary under that key. At the
   end, the dictionary values are the groups.

*Related problem families:*

- *Character counting / frequency analysis* (e.g., valid anagram,
  palindrome permutation)
- *Hash map grouping* (e.g., group by first letter, group by length)
- *String hashing* (e.g., using sorted strings, rolling hash)

*How to spot them:* If you see "group together items that are
rearrangements of each other", "same letters regardless of order" or
"items that can be transformed into each other by reordering", you're
likely looking at a canonical‑form + hash map problem.
