* next basic command

#+begin_src bash
next dev # : Starts the development server using Turbopack (default bundler).
next build #: Builds the application for production.
next start #: Starts the production server.
eslint # : Runs ESLint.
#+end_src

* The nextjs project structure in brief

Here are the key Next.js project structure elements:

*Core Directories & Files*
- =app/= (App Router) or =pages/= (Pages Router) - Main routing directories
- =public/= - Static assets like images, fonts, robots.txt
- =src/= (optional) - Source code can be placed here
- =components/= - Reusable React components
- =lib/= or =utils/= - Utility functions and libraries
- =styles/= - Global and modular CSS files

*Configuration Files*
- =next.config.js= - Next.js configuration
- =package.json= - Dependencies and scripts
- =.env= - Environment variables
- =tsconfig.json= - TypeScript configuration

*Common Files in =app/= Directory*
- =layout.jsx= - Shared layout component
- =page.jsx= - Page component for route
- =loading.jsx= - Loading UI
- =error.jsx= - Error boundary
- =route.js= - API route handlers
- =not-found.jsx= - 404 page

*Key Features*
- File-based routing (folders create URL paths)
- Server & Client components by default
- Built-in support for layouts, loading states, error handling
- Automatic code splitting and optimization
