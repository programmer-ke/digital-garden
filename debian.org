* debian

** package system
*** Structure of the binary package                                     :deb:
The =.deb= file format is an archive. It can be inspected with
=ar t foo.deb= and extracted with =ar x foo.deb=

It contains:
- =debian-binary= : a text file indicating package format version
- =control.tar.xz= : meta-information like name, version of package,
  scripts to install, uninstall, dependencies etc
- =data.tar.xz= : files to be extracted including executables,
  libraries and documentation

Content of the control file is similar to the output of =apt-cache show <package>=

The =alien= program can create .deb files from =.rpm= packages.

e.g. =fakeroot alien --to-deb phpMyAdmin-5.1.1-2.fc35.noarch.rpm=
*** Structure of a source package
A source package can be downloaded for example using apt: =apt source <package>=

Structure of a source package:
- =.dsc= file which shows author, list of files and their authenticity
- =.orig.tar.gz= source code as provided by original author
- =.debian.tar.xz= modifications made by the debiain maintainer,
  including a debian directory with instruction to construct the
  binary package(s)
  
*** dpkg                                                               :dpkg:

Used to handle debian packages. It is a system tool as opposed to apt
which is a user tool that complements it.

Some operations.

#+begin_src bash
dpkg -i man-db_2.9.4-2_amd64.deb # install

# unpack and configure
dpkg --unpack man-db_2.9.4-2_amd64.deb
dpkg --configure man-db

dpkg -r debian-cd  # removal

dpkg --listfiles package # (or -L), which lists the files installed by this package

dpkg --search file # (or -S), which finds the package(s) containing the file

dpkg --contents file.deb # (or -c), which lists the files in the Debian package specified;

dpkg --info file.deb #  (or -I), which displays the headers of this Debian package.

#+end_src


dpkg's log file is found at =/var/log/dpkg.log=

** Maintenance and Updates

*** sources.list

- The file =/etc/apt/sources.list= lists package source repositories.
  - Each line has at least 3 parts separated by spaces. Format is
    described =sources.list(5)=.
  - The first field describes the source type:
    - =deb= :: binary package source
    - =deb-src= :: source package
- =apt-cacher= and =apt-catcher-ng= can be used to proxy package
  sources and cache requests
- sources for external
    
*** aptitude, apt-get, apt

- =apt= and =apt-get= are build on the same library. =apt= is meant
  for interactive use. =apt-get= 's public interface is well defined
  and can be used in scripts.
- =aptitude= provides a gui.
- If a package is found in multiple distributions, a distribution be specified
  during installation e.g. =apt install spamassasin/unstable=

- History is tracked in the log file =/var/log/apt/history.log=  
- Packages installed only through dependencies are marked as auto, and
  can be automatically removed when the parent package is
  removed. This can be changed by marking them as manual

- =apt list ?automatic= lists automatic packages
- =apt list ?garbage= lists packages that can be auto removed

*** apt-cache

- Displays information stored in APT's internal cache, gathered from
  different sources in =sources.list= during =apt update=
  - The cache is found at =/var/lib/apt/lists=
- =apt-cache policy= displays pinning priorities of packages.

*** Checking package authenticity

- Package authenticity info is stored in files called =InRelease= and =Release= which
  are provided by debian mirrors.
- They contain a list of debian packages available on the repo along with their hashes and
  are cryptographically signed.
- apt needs a set of trusted GPG keys to verify the signatures, it gets them from
  files in =/etc/apt/trusted.gpg.d= and from the =/etc/apt/trusted.gpg= keyring
  managed by =apt-key= (deprecated)
  - 3rd party =.gpg= keys can be added to =/etc/apt/trusted.gpg.d=,
    however it is not clear which key matches which repository in this
    case, introducing vulnerabilities in case of malicious mirrors.
  - It is better to put 3rd party keys in =/usr/share/keyrings= and use the
    =Signed-By= field in the =sources.list= for 3rd party repositories
  - =.asc= files are discouraged, convert them to .gpg first (check whether they are armored or not)

*** Upgrading from one stable distribution to the next

- Read the release notes of the new version
- Tidy installed packages and only keep important ones
  - Tools that can help: aptitude, deborphan, debfoster, apt-show-versions, debsums
- To anticipate any problems, optionally install =apt-listchanges= and
  =apt-listbugs= to list problems before upgrade process begins
- Change =/etc/apt/sources.list= to get packages from the new repo.
- Update the database using =apt update=
  - In some cases fields in the =Release= and =InRelease= files may
    have changed. use the =--allow-releaseinfo-change= flag of
    =apt-get=
- Do a minimal upgrade with =apt upgrade=
  - ensure's package management tools have are up to date with bug fixes
- Do a full upgrade with =apt full upgrade=
  - Any unreported bugs can be reported using =reportbug=
- Post upgrade, find any packages that were removed (no package
  source) using =apt-show-versions | grep "No available version"=
  - could have been abandoned, or neglected with issues. Remove if not being used
- Some packages could have been renamed and the old name is retained
  as an almost empty package. These are known as dummy/transition
  packages.
  - They may be identified by tools like aptitude, deboprhan, or
    debfoster or =dpkg -l | grep ^ii | grep -i -E
    "(transition|dummy)"=
  - The new package is pulled in as a dependency of the transitional
    package, and may be removed if it is purged. A solution is to selectively install/remove e.g.
    - =apt install package1 package2-=
    - =apt remove package1+ package2=
- Configuration cruft can be purged using =apt autoremove --purge=
- =find= or =locate= can be used to manually remove configuration with
  leftover suffixes .dpkg-dist, .dpkg-old, .ucf-old.

* Keeping system up to date

- Controlled by variables in =/etc/apt/apt.conf.d/10periodic=
  - =APT::Periodic::Update-Package-Lists= :: frequency in days to
    refresh packages
  - =APT::Periodic::Download-Upgradeable-Packages= :: frequency in
    days of downloading packages
  - =APT::Periodic::AutocleanInterval=  :: How obsolete packages are
    removed from =apt-cache=
  - =APT::Periodic::Unattended-Upgrade= :: When enabled, the
    =unattended-upgrade= package will be used to automate the upgrade
    process. This can be configured using
    =/etc/apt/apt.conf.d/50unattended-upgrades= e.g. only download
    security updates.

- =apt-cache= allows searching packages by name and description
- =apt show <package>= allows one to get information about a known package
- =axi-search= allows additional criteria for searching packages e.g. files contained

* Solving problems and finding information

- Manual pages can be found using the =man= command
  - provided by the =man-db= package
  - =man man= gives info on man itself
  - To get a oneliner on a command, run =whatis <command>=
  - =apropos <keywords>= can be used to filter manpage summaries
  - the =man2html= package enables one to view manpages on a web browser

- Some gnu packages are documented more extensively using =info= documents
  - =info2www= provides a web interface

- Some packages can have documentation stored in READMEs under
  =/usr/share/doc/<package>=

- Some tips to get information on an unknown package:
  - Check =/usr/share/doc/package/README.Debian=
  - use =dpkg -L package= to list files in the package and identify available documentation
  - use =dpkg -s package= to show metadata
  - Check if there are examples in =/usr/share/doc/package/examples/=
    
- To monitor daemons:
  - Find log file under =/var/log/*=
  - rsyslogd is a special daemon that collects logs from other programs.
    - details are defined in =/etc/rsyslog.conf=

- Ask for help in mailinglists where possible

- Reporting bugs:
  - use =dpkg -S <file>= to find the package associated with a buggy program.
  - check the bugtracker for that package if the bug has been reported
  - send a bug report using the =reportbug= command
